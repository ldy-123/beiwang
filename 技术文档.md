```js

  id: "lx4k2abc",          // uid()生成，Date.now()+random
  title: "标题",
  description: "备忘内容", // 可选，最多100字，旧数据无此字段时视为空

    {
      id: "...",

      done: false,
      indent: 0,           // 0/1/2，层级缩进

  archived: false,
  archivedAt: null,        // 归档时间戳

  updatedAt: 1714000000000
}
```

## 关键实现细节（踩过坑，必须知道）

### 1. 左滑手势 vs CSS 入场动画的冲突

**问题**：`@keyframes cardIn` 最初用 `transform: translateY(10px)` 做入场效果，配合 `animation-fill-mode: both`。但左滑用的是 `card.style.transform = 'translateX(-Xpx)'`，JS 的 transform 会被 CSS animation 的 fill-mode 覆盖，导致左滑后卡片弹回去。

**修复**：把 `cardIn` 改成**纯 opacity 动画**，不动 transform：
```css
@keyframes cardIn {
  from { opacity: 0; }
  to   { opacity: 1; }
}
```
**绝对不能**在 `cardIn` 里加任何 transform，否则左滑会失效。

### 2. 新建标签取消选中后消失

**问题**：`renderTagSelector()` 的标签池来源是 `getAllTags()`（所有已保存备忘的标签）+ `editTags`（当前选中标签）。新建标签还没保存、一旦取消选中就从 `editTags` 移除，标签池里就没有它了，按钮消失。

**修复**：引入 `editAvailableTags` 独立维护标签池，在 `addTag()` 时同时 push 进去，`renderTagSelector()` 改为：
```js
const all = [...new Set([...getAllTags(), ...editAvailableTags])];
```
`editAvailableTags` 在 `openNew()` / `openEdit()` 时初始化，整个编辑会话内只增不减。

### 3. Service Worker 缓存版本

`sw.js` 第一行的 `CACHE` 版本号控制缓存更新。**每次修改任意文件后必须更新版本号**，否则用户拿到的仍是旧缓存：
```js
const CACHE = 'memo-v23'; // 改完代码就递增这里
```
当前最新是 `v30`。

### 4. 切换标签页时左滑按钮闪烁

**问题**：`cardIn` 入场动画挂在 `.memo-card` 上（opacity 0→1），但 `.swipe-actions` 是 `.swipe-item` 里的兄弟元素，不受动画控制。卡片透明期间，后面的按钮透出来形成闪烁。

**修复**：把 `animation: cardIn` 和 `animation-delay` 从 `.memo-card` 移到 `.swipe-item`，整个容器一起淡入，按钮永远不会透出。

### 5. 标签栏拖拽排序：无法拖到最后一位

**问题**：`_findTagChipAt(x)` 在 x 超过最后一个 chip 右边缘时返回 `null`，导致无法把标签拖动到末尾。

**修复**：检测 `x > lastChip.right`，直接返回 `tagOrder.length`（追加到末尾），同时给最后一个 chip 加 `.tag-drop-after`（右侧竖线）而非 `.tag-drop-here`（左侧竖线）。

### 6. 待办拖拽层级约束

**规则**：子节点缩进不能超过上方节点的层级 + 1；如果插入到最顶部（没有上方节点），只能是顶级（indent = 0）。

**实现**：`_updateDrag()` 在计算目标 indent 前，先扫描得到插入点正上方那条待办的 `indent`：
```js
const maxIndent = aboveIndent < 0 ? 0 : Math.min(2, aboveIndent + 1);
const indent = Math.min(rawIndent, maxIndent);
```
`aboveIndent = -1` 表示没有上方节点。

### 7. 待办拖拽时实时显示目标缩进

**问题**：原拖拽只有 drop-line 指示位置，看不到落下后的缩进层级。

**修复**：
- ghost 横向跟随拖动：`ghost.style.left = ghostBaseLeft + indent * 22 + 'px'`；`ghostBaseLeft` 在 `_initDrag` 时存为 `rect.left - currentIndent * 22`（剥离当前缩进，确保 indent=0 时对齐最左）
- drop-line 加粗（3px）并在左端加圆点（`::before`），视觉上指示缩进起点

### 8. 日期选择器底部按钮超出视口

**问题**：`_dpRender()` 里固定用 `top = r.bottom + 6` 把选择器放在锚点下方，当锚点在屏幕中部偏下时，选择器底部（时间行 + 确认按钮）超出视口，无法点击。

**修复**：先算选择器高度，空间不足时改为弹到锚点上方：
```js
const H = wrap.scrollHeight || 420;
let top = r.bottom + 6;
if (top + H > window.innerHeight - 8) {
  top = Math.max(8, r.top - H - 6);
}
```

## 样式系统

主题用 CSS 变量，定义在 `:root`，浅色主题，无暗色模式。

**主要色板**：
- `--amber: #f59e0b` — 主色，FAB、选中状态、保存按钮
- `--teal: #6366f1` — 辅色（实际是靛蓝），次要交互
- `--bg: #f4f5fa` — 页面背景
- `--surface: #ffffff` — 卡片背景
- `--text: #1c1d2e` — 主文字

**背景效果**：`body::before` 用 3 层 radial-gradient 制造琥珀/靛蓝光晕 + 26px 点阵纹理。

## 已知限制 / 注意事项

1. **数据不同步**：各自存在各自手机的 localStorage，无云端，无多设备同步
2. **换机迁移**：通过导出/导入 JSON 手动迁移
3. **清除浏览器数据会丢失所有备忘**（PWA 的 localStorage 也会被清掉）
4. **iOS Safari** 对 PWA 支持有限，但基本功能可用；iOS 的「添加到桌面」入口在 Safari 分享菜单里
5. **manifest.json 的 id 字段**设为 `/beiwang/beiwang/`，与部署路径匹配，不能改动，否则重新安装会视为新 App，丢失 PWA 状态
