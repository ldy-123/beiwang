# 随手记 PWA — 技术交接文档

## 项目概况

一个纯静态、无框架的移动端随手记 PWA，已部署到 GitHub Pages。

- **线上地址**：`https://ldy-123.github.io/beiwang/`
- **仓库**：`https://github.com/ldy-123/beiwang`
- **技术栈**：原生 HTML / CSS / JavaScript，Supabase JS SDK（CDN 加载），无其他第三方依赖
- **数据存储**：优先 localhostStorage（离线可用），登录后同步到 Supabase（云端备份 + 多设备同步）

## 架构总览：三大模块

应用有三个平行的数据模块，各自遵循相同模式：

| 模块 | 存储 Key | 状态变量 | 列表渲染 | 弹窗 ID |
|------|---------|---------|---------|--------|
| 备忘 | `memos_v1` | `state` | `renderList()` | `overlay` |
| 小记 | `notes_v1` | `noteState` | `renderNoteList()` | `noteOverlay` |
| 打卡 | `habits_v1` | `habitState` | `renderHabitList()` | `habitOverlay` |

三个模块共用 `#memoList` DOM 元素渲染列表。`renderAll()` 是中央调度器，根据 `state.currentTab` 决定渲染哪个模块。跨模块修改数据后始终调用 `renderAll()`。

## Supabase 云同步

应用支持可选的云同步功能，基于 Supabase（开源 Firebase 替代）。

### 架构

- **认证**：Supabase Auth，邮箱 + 密码方式，支持注册、登录、找回密码、修改密码
- **数据表**：memos / notes / habits 三张表，行级安全（RLS）策略确保每个用户只能访问自己的数据
- **同步方式**：登录时全量拉取（`syncAllFromCloud()`），保存时全量推送（批量 upsert），删除时即时从云端移除
- **离线降级**：未登录或网络断开时，应用退化为纯 localhostStorage 模式，不影响正常使用
- **实时订阅**：`supabase_realtime` publication 包含三张表，预留实时多设备同步能力

### 前端实现

```js
// 初始化
const SUPABASE_URL = 'https://xxx.supabase.co';
const SUPABASE_KEY = 'sb_publishable_xxx';
sb = supabase.createClient(SUPABASE_URL, SUPABASE_KEY);

// 登录 → 同步
sb.auth.onAuthStateChange((event, session) => {
  if (event === 'SIGNED_IN') syncAllFromCloud(); // 拉取全量数据
});

// 保存 → 推送到云
function save() {
  localStorage.setItem(...);                   // 本地持久化
  if (sb && sbAuthed) _syncTableToCloud('memos'); // 云端同步
}
```

- `_sbRow(table, item)` — 将前端数据模型转为 Supabase 行格式
- `_sbDelete(table, id)` — 从 Supabase 删除指定行
- `_syncTableToCloud(table)` — 将当前 state 全量批量 upsert 到 Supabase
- `syncAllFromCloud()` — 登录后从 Supabase 拉取三表全量数据，覆盖本地 state 和 localStorage

### 部署需提供

- Supabase 项目 URL
- Supabase anon key（`sb_publishable_` 开头，非 secret key）
- 数据库表已通过 SQL 建好并启用 RLS

## key 实现细节

### 左滑手势 vs CSS 动画

- `cardIn` 动画只能用 opacity，不能用 transform，否则 JS 的 `translateX` 会被 CSS animation fill-mode 覆盖
- `animation: cardIn` 挂在 `.swipe-item` 而非 `.memo-card`，防止卡片透明期间后面的按钮透出
- 各模块有独立滑动实现：`initSwipes` / `initNoteSwipes` / `initHabitSwipes`，模式相同

### 标签拖拽

- 触发：移动端长按 300ms，桌面端 pointerdown + 移动阈值 6px
- CSS 三重防护防止 Android 系统弹出菜单：`user-select: none; -webkit-touch-callout: none; touch-action: manipulation`
- 再加 `contextmenu` preventDefault
- 用 `pointerdown` 而非 `mousedown`，加 `e.pointerType !== 'mouse'` 判断，避免移动端拦截 click
- 排序持久化到 `tagOrder_v1`

### 小记全屏编辑器

- 全屏模式（`.modal.note-full`），`height: 100dvh`，键盘弹起时自动适配
- 统一编辑区：标题 input（无边框，placeholder "标题"）+ 时间分隔（`fmtDate` 格式）+ contenteditable div
- 工具栏移至底部（`.note-toolbar`），颜色面板向上弹出防键盘遮挡
- 键盘适配：`visualViewport` resize 监听 + toolbar padding 动态调整
- 加粗：`execCommand('bold')`
- 下划线：`execCommand('underline')`
- 颜色：`execCommand('foreColor', false, color)`，8 色下拉盘
- 图片：base64 插入 `<img>`
- 编辑器 blur 时保存 Selection，工具栏按钮 mousedown 时恢复，解决焦点丢失问题

### 待办拖拽

- 层级约束：子节点缩进 ≤ 上方节点层级 + 1，最顶只能是顶级
- ghost 实时反映缩进：`ghostBaseLeft` 存基准位置，拖拽时加 `indent * 22` 像素
- drop-line 加粗 + 左端圆点指示缩进起点

### 日期选择器

- 锚点下方弹出，空间不足自动翻到上方
- 视觉风格为 Linear/Raycast 极简工具风，绿色主色(#15803d)

### 小记⋯编辑菜单

- 每条小记卡片右上角 `⋯` 按钮，点击弹出「✏️ 编辑」选项
- 全局共用一个 `#noteCardMenu` 下拉实例（`initNotes()` 中创建）
- 点击按钮时根据按钮位置动态定位下拉菜单
- 菜单按钮 `mousedown`/`touchstart`/`click` 均 `stopPropagation`，防止触发展开/收起和长按编辑
- `_bindNoteCardMenu()` 在每次 `renderNoteList()` 和 `renderArchiveAll()` 渲染后重新绑定
- 点击菜单外区域自动关闭

### 通知

- 待办到期时间通过 `setTimeout` 触发 Web Notification API
- 最长延迟 24 小时（浏览器限制）
- 页面回到前台时（`visibilitychange`）重建定时器并补发 24h 内过期的通知
- 通知权限请求已移到保存操作的用户手势上（iOS 要求在用户手势下调用 `requestPermission()`）
- 去重机制：`_notifFired` 防止同一待办重复弹通知
- 每次 `save()` 重建所有定时器
- PWA 安装后支持后台通知（iOS 16.4+）

### Service Worker

- network-first 策略，在线始终拉取最新，离线降级用缓存兜底
- 修改 JS/CSS/HTML 后需递增 `CACHE` 版本号，确保 install 事件拉取新资源缓存（离线兜底用）

## 已知限制

1. 未登录时数据仅存 localStorage，清除浏览器数据会丢失（登录后云端自动备份）
2. iOS PWA 限制：通知需要 iOS 16.4+ 且已安装到桌面
3. `manifest.json` 的 `id` 字段为 `/beiwang/`，与部署路径匹配，不可改动
4. `setTimeout` 通知在页面后台时不可靠，已通过前台恢复补发机制缓解
